## The Shift We're Riding

There is a quote circulating in the AI space right now that frames exactly why Agent Hub matters:

> **"In the future, agencies and companies won't be using agent services — they'll be running their own agent teams."**

This is already happening. OpenClaw, Claude Code, Codex — teams are deploying AI agents inside their own infrastructure, in-browser or in-cloud. The question nobody has answered yet is:

**When you're running a team of agents, how do you actually manage them?**

You can see the agents. You can feel them working. But where's the dashboard? Where's the visibility? Where's the control layer?

That gap is Agent Hub.

---

## Core Positioning Statement

> **Agent Hub is the management layer for your AI agent team.**

Not a coding tool. Not just a file editor. The place where you see your whole agent operation in one view — who's running, what they know, what skills they have, what context they're working from.

Think of it as the operations interface for your agent workforce.

---

## Why Platform-Agnostic Is Non-Negotiable

OpenClaw is the current standard we're building on. But it won't be the only one forever. Standards shift. New frameworks emerge. The multi-agent ecosystem is still being defined.

**Agent Hub must work with whatever comes next.**

This isn't just technical hedging — it's the marketing message. We're not building an OpenClaw plugin. We're building the management infrastructure layer that sits above any agent system. OpenClaw today. Whatever else tomorrow.

Copy angle: *"Works with OpenClaw. Ready for whatever comes next."*

## ICP — Who This Is For

### Current PRD personas (keep these — they're right):
- AI Engineer running OpenClaw/Claude Code
- Vibe Coder with skills scattered everywhere
- AI PM/Builder managing agent fleets

### Expanded ICP — the "creative-first organization":
The opportunity to position Agent Hub is beyond individual developers. The real emerging market is **organizations that are starting to run agent teams internally** — and notably, these are disproportionately *creative-first organizations*:

- **Creative agencies** — managing AI agents for content, design, research
- **Animation and film studios** — AI pipelines for production workflows
- **Music and audio studios** — using agents for marketing, distribution, research
- **Digital marketing agencies** — running specialized agents per platform/task
- **Boutique design shops** — AI-native design and development workflows
- **Creator-led businesses** — mental health, coaching, spiritual, education

What these organizations share: they approach their work as *craft*, not just commerce. They care deeply about the quality and coherence of their agent operations — not just "does it work" but "does it reflect who we are." Agent Hub gives them the clarity to maintain that coherence at scale.

**This is the segment to call out on the landing page.** Not just solo developers — teams. Not just tech companies — creative organizations that have made the leap to running agent teams.

## The Problem Reimagined

The current PRD frames the problem around scattered files and bad editors. That's correct for the developer persona. For the broader ICP, frame it bigger:

### Developer-level problem (keep):
Context is scattered. Skills are chaos. Wrong tools.

### Organization-level problem (add):
You've built an agent team. Now what? You can't see it. You can't manage it. You don't know if your agents have the right context to do their jobs. You don't know which agent has which skill. Someone updates a SOUL.md and nobody knows.

**Running an agent team without Agent Hub is like running a company without an org chart, an onboarding doc, or a project board.**

## What NOT to Say

- Don't position it as an OpenClaw-exclusive tool. Mention OpenClaw compatibility — don't make it the whole story.
- Don't oversell enterprise — the free/open-source angle is the right trust builder right now. Let the community grow first.
- Don't use generic AI marketing language: "powerful," "seamless," "next-generation." Developers see through it.
- Don't obscure that it's a file editor at its core. The honesty is the differentiator.

## UPDATE — 2026-03-03 (afternoon sprint)

### The three lego paradigms — add to landing page

This framing came out of a product session and should be the core structural metaphor on the landing page and in all copy. Use this verbatim or close to it:

> **Three building blocks. One system.**
>
> **Skills** — reusable capabilities you drop onto any agent. Like npm packages for your team.
>
> **Agents** — your team members. Each has their own skill set, personality, and context.
>
> **Context & Files** — the instructions, memory, and PM files that shape how each agent thinks and acts.

Everything in Agent Hub maps to one of these three paradigms. This is the mental model users need to learn once, then everything makes sense.

---

### The three views — connect to paradigms in copy

| View | Paradigm it serves | What you do |
|---|---|---|
| **Files** | Context & Files | Edit who an agent is and what they know |
| **Crons** | Automation | Schedule what runs automatically |
| **Canvas** *(coming)* | Agents + Skills | See and shape the whole team visually |

Use this in the features section. Each view card should show its paradigm label.

---

### The Canvas — headline feature copy

The Canvas view is the defining product feature. Lead with this:

> **Lists are for managing. A canvas is for thinking.**
>
> You can't understand a team in a spreadsheet. On the Canvas, you see every agent as a card — who they are, what skills they carry, what context they share. Drag a skill onto an agent. Connect shared context. Add a new agent in one click.
>
> Your agent org chart, alive and editable.

### The target user — sharpen

The user is a **solo founder or independent creator who manages 10 agents instead of 10 people.** They're making the same strategic orchestration decisions a manager makes — but for an AI team:

- Which agent owns which domain?
- What skills does each agent need?
- What shared context does the whole team need to know?
- How do I onboard a new agent without it going off-script?

These are not technical questions. They're management questions. Agent Hub is the management interface.
